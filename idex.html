THEORY & DEFINITIONS
PID CONTROLLERS

What is a Controller?

A controller is a device that compares the measured process variable (PV) to the desired setpoint (SP), computes an error ε(t) = SP − PV, and generates a corrective output signal to eliminate that error. In process control, the controller output drives a Final Control Element (FCE — usually a valve) to manipulate the process.

ERROR SIGNAL

ε(t) = SP − PV — the difference between what we want and what we have. Positive error means PV is below SP; controller must increase output.

P, PI, PD, PID — Controller Types

P (Proportional): Output proportional to current error. Fast but leaves a permanent offset.

PI (Proportional-Integral): Adds integral action — eliminates steady-state offset by accumulating error over time. Most common in industry.

PD (Proportional-Derivative): Adds derivative action — anticipates future error. Reduces overshoot but amplifies noise.

PID: All three combined — best overall performance for most processes.

PID Mathematical Equations

P Controller: p(t) = Kc · ε(t) + ps
// ps = bias (steady-state output), Kc = proportional gain
P(s)/ε(s) = Kc // Transfer Function of P controller
PI Controller: p(t) = Kc·ε(t) + (Kc/τI)·∫ε(t)dt + ps
p(t) = Kc·ε(t) + KI·∫ε(t)dt // where KI = Kc/τI

P(s)/ε(s) = Kc(1 + 1/τIs) = Kc·(τIs + 1)/(τIs) // TF of PI
// τI = integral time / reset time (minutes)
// 1/τI = reset rate (repeats/minute)
PID Controller: p(t) = Kc·ε(t) + (Kc/τI)·∫ε(t)dt + Kc·τD·(dε/dt) + ps
p(t) = Kc·ε(t) + KI·∫ε(t)dt + KD·(dε/dt) + ps

P(s)/ε(s) = Kc(1 + 1/τIs + τDs) // TF of PID

Proportional Band (PB)

PB% = (100 / Kc) → Kc = 100 / PB%
// PB% = % of measurement range that gives full output change
// Narrow PB (small %) = high gain = aggressive control
// Wide PB (large %) = low gain = sluggish control

Alternatively: PB% = (Δinput / full measurement range) × 100

Key Concepts: Anti-Reset Windup (ARW)

When the actuator (valve/heater) saturates at its limit, the controller cannot respond further — but the integral term keeps accumulating error ("winds up"). When the process finally responds, the controller output is already excessively large → huge overshoot, slow recovery, poor stability.

ARW solution: Stop integrating when the output is saturated. Modern controllers implement this automatically.

Derivative Kick — Problem & Solution

When SP changes suddenly (step), dε/dt → ∞ (ideally), causing a huge spike in derivative output — the "derivative kick." This can damage actuators and cause instability.

Solution — Derivative on Measurement (DOM): Instead of differentiating error ε = SP − PV, only differentiate the process variable PV:

Standard: D_out = KD · dε/dt = KD · d(SP-PV)/dt
Modified: D = −KD · d(PV)/dt // assumes dSP/dt = 0
// This eliminates kick caused by SP step changes
// D still reacts to PV changes (process dynamics)

Final Modified PID: y(t) = y_bias + Kp·ε(t) + KI·∫ε(t)dt − KD·d(PV)/dt

Manual PID Tuning Algorithm

MANUAL TUNING PROCEDURE (ONLINE SYSTEM)
Set KI = 0 and KD = 0 (P-only control first)
Increase Kp gradually until the output oscillates continuously → record this as Kp_critical
Set Kp = Kp_critical / 2 for a "quarter-amplitude decay" response
Increase KI slowly until steady-state offset is corrected within acceptable time — too large KI causes instability
Increase KD if needed to reduce overshoot and speed up recovery after disturbances
Verify: too much Kp → excessive overshoot; if oscillatory → reduce KD; if cannot track → increase Kp and KI

THEORY & FORMULAS
2ND ORDER SYSTEMS

Standard 2nd Order Transfer Function

G(s) = K·ωn² / (s² + 2ζωn·s + ωn²)
// Equivalently, in time-constant form: G(s) = K / (τ²s² + 2ζτs + 1)

// Where:
K = steady-state gain
τ = natural period (τ = 1/ωn)
ζ = damping ratio (zeta)
ωn = natural frequency (rad/time)

Extracting Parameters from TF

Given: G(s) = A / (as² + bs + c)

Divide by c: G(s) = (A/c) / [(a/c)s² + (b/c)s + 1]

→ K = A/c
→ τ² = a/c → τ = √(a/c)
→ 2ζτ = b/c → ζ = (b/c) / (2τ)
→ ωn = 1/τ
→ ωd = ωn·√(1−ζ²) // damped natural frequency

Damping Types

ConditionTypeResponse Behaviour
ζ = 0UndampedOscillates forever, no decay
0 < ζ < 1UnderdampedOscillates with decaying amplitude
ζ = 1Critically dampedFastest approach to SS without overshoot
ζ > 1OverdampedSlow approach, no oscillation, no overshoot

Complete Performance Formulas (Underdamped, 0 < ζ < 1)

ωd = ωn·√(1 − ζ²) // damped frequency

%OS = exp(−πζ / √(1−ζ²)) × 100 // percent overshoot
DR = (%OS/100)² // decay ratio = OS²
T = 2π / ωd // period of oscillation
tp = π / ωd // time to first peak
tr = (π − arccos ζ) / ωd // rise time
ts = 4 / (ζ·ωn) // settling time (2% criterion)

y(∞) = K × magnitude of step // ultimate value
y_max = y(∞) × (1 + %OS/100) // maximum value

Pole-Zero Analysis

For the standard 2nd order TF, poles are found from the characteristic equation: τ²s² + 2ζτs + 1 = 0

s = [−2ζτ ± √(4ζ²τ² − 4τ²)] / (2τ²)
s = −ζωn ± jωn√(1−ζ²)
s = −ζωn ± jωd

// For underdamped: poles are complex conjugates in LHP → stable, oscillatory
// Real part = −ζωn (decay rate), Imaginary part = ωd (oscillation frequency)
// System is STABLE if all poles have NEGATIVE real parts (LHP)

BLOCK DIAGRAM REDUCTION

Block Diagram Elements

  • Block (G(s)): Transfer function — relates output to input: C(s) = G(s)·R(s)
  • Summing Point (⊗): Algebraically adds/subtracts signals; (+) adds, (−) subtracts
  • Branch/Pickoff Point (•): A point from which the same signal splits to go to multiple blocks
  • Signal Arrow (→): Direction of signal flow; signals travel one way only
  • Forward Path: Path from input R(s) to output C(s) through G(s)
  • Feedback Path: Path from output C(s) back to summing point through H(s)

Key Terminology

CLOSED-LOOP TF (Control Ratio)

C(s)/R(s) = G(s) / [1 ± G(s)H(s)]

— Negative feedback: 1 + GH  |  Positive feedback: 1 − GH

OPEN-LOOP TF

G(s)·H(s) — product of forward and feedback TFs

CHARACTERISTIC EQUATION

1 + G(s)H(s) = 0 — setting denominator of CL TF to zero; roots determine system stability

ACTUATING SIGNAL (ERROR)

E(s) = R(s) ∓ H(s)C(s) — the signal that drives the controller

The 6 Reduction Rules

BLOCK DIAGRAM REDUCTION — STEP-BY-STEP ALGORITHM
Combine all cascade (series) blocks: G_eq = G1 × G2 × G3 × ...
Combine all parallel blocks: G_eq = G1 ± G2 ± G3 ± ...
Eliminate all minor (inner) feedback loops: G_eq = G / (1 + GH) for each inner loop
Shift summing points to the left if needed (multiply the branch by 1/G when crossing G forward)
Shift takeoff points to the right if needed (multiply the branch by G when moving a pickoff across G forward)
Repeat steps 1–5 until the entire diagram reduces to a single block C(s)/R(s)
#OperationRule / EquationNote
1 Series blocks G_eq = G1·G2·G3 Multiply all TFs
2 Parallel blocks G_eq = G1 ± G2 Sum (+ add) or difference (− subtract)
3 Feedback loop G_eq = G / (1 + GH) Negative feedback. For positive: 1−GH
4 Move pickoff BEHIND G Add 1/G in the moved branch Branch must still carry same signal value
5 Move pickoff AHEAD of G Add G in the moved branch Branch must still carry same signal value
6 Move summing BEHIND G Add G in the moved branch u2 branch gets multiplied by G
7 Move summing AHEAD of G Add 1/G in the moved branch u2 branch gets divided by G

Flowchart — How to Reduce Any Block Diagram

START: Draw/label all blocks, signals, summing & pickoff points │ ▼ Are there series blocks? YES → Multiply them → Single block NO → Continue │ ▼ Are there parallel paths? YES → Sum them (±) → Single block NO → Continue │ ▼ Are there minor/inner feedback loops? YES → Apply: G/(1+GH) → Eliminate inner loop NO → Continue │ ▼ Are there tangled summing/pickoff points blocking simplification? YES → Move summing LEFT or pickoff RIGHT (per Rules 4−7) NO → Continue │ ▼ Is diagram reduced to single loop? NO → Repeat from top YES → Apply final feedback rule: C/R = G/(1+GH) │ ▼ DONE: Single block with overall TF = C(s)/R(s)

SOLVED PROBLEM
TWO TANKS IN SERIES (NON-INTERACTING)
PROBLEM STATEMENT

Two tanks drain independently but in series. Outflow of Tank 1 enters Tank 2 without hydraulic interaction.

(a) Show why the TF becomes a product of two first-order systems.

(b) Obtain the overall 2nd order TF. Comment on overshoot/oscillations.

Assumptions
  • Tanks are non-interacting: outflow of Tank 1 = h₁/R₁, independent of h₂
  • Linear resistance to flow: Q_out = h/R (gravity-driven outflow)
  • Constant cross-sectional areas A₁ and A₂
  • Laplace transform with zero initial conditions
Equations Used & Why

We apply mass balance (conservation of mass) to each tank: Accumulation = In − Out. For liquid systems, volume = A·h, so d(Ah)/dt = Q_in − Q_out. This gives a 1st order ODE for each tank. Taking Laplace yields a 1st order TF per tank. Since the tanks are non-interacting, the TFs multiply (series connection in block diagram).

Calculations — Part (a)
Tank 1 — Mass balance: A₁ dh₁/dt = Fin(t) − h₁/R₁ Taking Laplace (zero IC): A₁s·H₁(s) = Fin(s) − H₁(s)/R₁ H₁(s)[A₁s + 1/R₁] = Fin(s) H₁(s)/Fin(s) = R₁/(A₁R₁s + 1) = R₁/(τ₁s + 1) where τ₁ = A₁R₁ Tank 2 — Input is outflow of Tank 1 = h₁/R₁: A₂ dh₂/dt = h₁/R₁ − h₂/R₂ Taking Laplace: A₂s·H₂(s) = H₁(s)/R₁ − H₂(s)/R₂ H₂(s)/H₁(s) = R₂/(τ₂s + 1) where τ₂ = A₂R₂ Overall TF: G(s) = H₂(s)/Fin(s) = [H₁/Fin] × [H₂/H₁] G(s) = R₁R₂ / [(τ₁s + 1)(τ₂s + 1)] ← Product of two 1st order TFs

Since Tank 2 doesn't influence Tank 1 (no back-pressure), the dynamics are completely independent, and the block diagram shows two TFs in series → overall TF = product.

Calculations — Part (b) Overshoot Analysis
Expand the denominator: (τ₁s+1)(τ₂s+1) = τ₁τ₂s² + (τ₁+τ₂)s + 1 Compare with standard 2nd order form: τ²s² + 2ζτs + 1 → τ² = τ₁τ₂ → τ = √(τ₁τ₂) → 2ζτ = τ₁ + τ₂ → ζ = (τ₁ + τ₂) / (2√(τ₁τ₂)) Proof that ζ ≥ 1 (AM ≥ GM inequality): Arithmetic Mean: (τ₁ + τ₂)/2 Geometric Mean: √(τ₁τ₂) By AM-GM: (τ₁+τ₂)/2 ≥ √(τ₁τ₂) for all τ₁,τ₂ > 0 Therefore: (τ₁+τ₂)/(2√(τ₁τ₂)) ≥ 1 ∴ ζ ≥ 1 always → System is OVERDAMPED (or critically damped)
Final Answer
ANSWERS — TWO TANKS

(a) Because the tanks are non-interacting (hydraulically independent), the TF of Tank 2 doesn't depend on what happens in Tank 2 when calculating Tank 1's TF. The two blocks are in series: G(s) = G₁(s) · G₂(s) = R₁R₂/[(τ₁s+1)(τ₂s+1)].

(b) Overall TF: G(s) = R₁R₂ / [(τ₁s+1)(τ₂s+1)] — this is 2nd order with ζ = (τ₁+τ₂)/(2√τ₁τ₂) ≥ 1 always (by AM-GM inequality).

Conclusion on Overshoot: Since ζ ≥ 1 always, the system is overdamped or critically damped. There is NO overshoot and NO oscillation — the step response rises monotonically in an S-curve to steady state.

I-term = Kc·(1/τI)·∫ε dt — accumulates error over time; eliminates offset but adds phase lag.

D-term = Kc·τD·(dε/dt) — reacts to rate of error change; anticipates future error.

Final Answer

  • Proportional Output = 20 units
  • Integral Output = 40 units
  • Derivative Output = 4 units

PROBLEM STATEMENT

Derive the transfer function of a PI controller. Estimate the dynamic response of PI controller output to a step change in error. Plot the results over 5 consecutive reset time intervals.

Derivation of PI Transfer Function

Why use Laplace? To convert the integral in the time-domain equation into an algebraic expression (multiplication by 1/s), enabling transfer function analysis.

PI time-domain equation: p(t) = Kc·ε(t) + (Kc/τI)·∫₀ᵗε(t')dt' + ps Taking Laplace (zero IC, ps = 0): P(s) = Kc·E(s) + (Kc/τI)·E(s)/s P(s) = Kc·E(s)[1 + 1/(τI·s)] Gc(s) = P(s)/E(s) = Kc·(τIs + 1)/(τIs) This has: One ZERO at: s = −1/τI (provides lead action) One POLE at: s = 0 (pure integrator → eliminates steady-state offset)
Step Response Derivation

For a step change of magnitude A in error: E(s) = A/s

P(s) = Gc(s) × E(s) = Kc·(τIs+1)/(τIs) × A/s P(s) = Kc·A·(τIs+1) / (τIs²) Expand: P(s) = KcA/s + KcA/(τIs²) Inverse Laplace (using: L⁻¹{1/s}=1, L⁻¹{1/s²}=t): p(t) = KcA + (KcA/τI)·t for t ≥ 0⁺ This consists of: Term 1: KcA → Instantaneous jump = proportional action (at t=0⁺) Term 2: (KcA/τI)·t → Linear ramp = integral action (grows with time)

PI CONTROLLER TF AND STEP RESPONSE

Transfer Function: Gc(s) = Kc·(τIs + 1)/(τIs)

Step response: p(t) = KcA + (KcA/τI)·t

At t = 0⁺: output jumps instantly by KcA (proportional kick). Then rises linearly. After each reset time τI, the integral adds another KcA to the output.

PNEUMATIC PROPORTIONAL CONTROLLER
PROBLEM STATEMENT

A pneumatic proportional controller controls temperature in range 60–100°F. Output pressure goes from 3 psi (valve fully open) to 15 psi (valve fully closed) as temperature goes from 71°F to 75°F with setpoint held constant. Find the gain and proportional band.

Assumptions
  • Linear relationship between measured temperature and controller output pressure
  • SP is constant → all input change is from the measured variable (same as error change)
  • Full output span = 3 to 15 psi = 12 psi (standard pneumatic signal range)
Given Data
Total measurement range = 100 − 60 = 40 °F Output span = 15 − 3 = 12 psi Input change causing full output = 75 − 71 = 4 °F Output change = 15 − 3 = 12 psi
Equations Used & Why

Gain Kc = ΔOutput / ΔInput — controller gain is the slope of the output-vs-input curve (units: psi/°F here). This tells us how much the output changes per unit change in measured variable.

PB% = (Input span causing full output / Total measurement range) × 100 — expresses the proportional band as a percentage of the total instrument range.

Calculations
Gain: Kc = ΔOutput / ΔInput = (15 − 3) psi / (75 − 71) °F = 12 / 4 = 3 psi/°F Proportional Band: PB% = (Input range for full output change / Total measurement range) × 100 = (4 °F / 40 °F) × 100 = 10 % Verification using PB = 100/Kc (normalized): Kc (normalized, %output/%input) = (12/12 × 100) / (4/40 × 100) = 100% / 10% = 10 PB = 100/10 = 10% ✓
Final Answer
ANSWERS — PAGE 129

Controller Gain Kc = 3 psi/°F

Proportional Band PB = 10%

Interpretation: A 10% PB means the controller output goes from 0% to 100% for only 10% of the measurement span (4°F out of 40°F). This is a high-gain, aggressive controller.

PROBLEM STATEMENT

PID controller at steady state with output = 9 psig. At t = 0, setpoint moves away from pen point at 0.5 in/min in direction of lower readings.

Settings: Kc = 2 psig/in, τI = 1.25 min, τD = 0.4 min. Plot output vs time.

Assumptions
  • Pen point (measured value) stays constant; SP moves away at 0.5 in/min in "lower" direction

PROBLEM STATEMENT

  • Unity feedback H(s) = 1
  • Zero initial conditions
  • Standard negative feedback closed-loop: T(s) = L(s)/[1+L(s)]
  • Performance metrics derived from CL characteristic equation
Step-by-Step Solution

(a) Open-Loop and Closed-Loop TF

C(s) = 3 + 2/s + 1·s = (3s + 2 + s²)/s = (s² + 3s + 2)/s Open-loop: L(s) = C(s)·Gp(s) = [(s²+3s+2)/s] × [2/(4s+1)] = 2(s²+3s+2) / [s(4s+1)] = 2(s+1)(s+2) / [s(4s+1)] Closed-loop: T(s) = L(s) / [1 + L(s)] Denominator of T(s): s(4s+1) + 2(s²+3s+2) = 4s² + s + 2s² + 6s + 4 = 6s² + 7s + 4 T(s) = 2(s²+3s+2) / (6s²+7s+4)

(b) Steady-State Error

For STEP input: E(s) = R(s) - C(s) = R(s)[1 - T(s)] SSE (step) = lim(s→0) s·E(s) = lim(s→0) s·(1/s)·[1-T(0)] T(0) = 2(0+0+2)/(0+0+4) = 4/4 = 1 SSE (step) = 1 - 1 = 0 // PID has integrator (1/s in KI term) → type-1 system → zero SSE for step ✓ For RAMP input R(s) = 1/s²: Velocity constant Kv = lim(s→0) s·L(s) = lim(s→0) 2(s²+3s+2)/(4s+1) = 2(0+0+2)/(0+1) = 4 SSE (ramp) = 1/Kv = 1/4 = 0.25 units

(c) Rise Time & Settling Time from CL Poles

CL denominator: 6s² + 7s + 4 = 0 Divide by 6: s² + (7/6)s + (4/6) = 0 s² + 1.1667s + 0.6667 = 0 Compare: s² + 2ζωns + ωn² = 0 ωn² = 0.6667 → ωn = 0.8165 rad/s 2ζωn = 1.1667 → ζ = 1.1667/(2×0.8165) = 0.714 // ζ = 0.714 → underdamped ωd = ωn√(1-ζ²) = 0.8165×√(1-0.510) = 0.8165×0.700 = 0.572 rad/s Rise Time: tr = (π - arccos ζ)/ωd = (3.1416 - arccos 0.714)/0.572 = (3.1416 - 0.775)/0.572 = 2.367/0.572 = 4.14 s Settling Time: ts = 4/(ζ·ωn) = 4/(0.714×0.8165) = 4/0.583 = 6.86 s

(d) Effect of Derivative Term (KD=1)

%OS = exp(-πζ/√(1-ζ²)) × 100 = exp(-π×0.714/0.700) × 100 = exp(-3.207) × 100 = 4.0 % Without KD (PI only, KD→0): New denominator = 6s² + (7-6)s + 4 = 6s² + s + 4 (approx) → ζ_PI ≈ 1/(2√24) ≈ 0.102 → OS_PI ≈ 72 % // Conclusion: KD dramatically increases damping from ζ≈0.1 to ζ=0.714 // Derivative term reduces overshoot from ~72% → ~4% !

DISTURBANCE REJECTION — REGULATOR ANALYSIS
PROBLEM STATEMENT

Pressure control system with: Gp(s) = 1/(8s+1), Gd(s) = 1/(4s+1), Kp = 2 (P-only controller).

Disturbance of +10 kPa enters at t = 0. Find: immediate effect, final pressure deviation, whether SSE exists, speed of rejection.

Assumptions
  • Unity feedback and unity measuring device (Gm = 1)
  • P-only controller: Gc(s) = Kp = 2
  • Disturbance D(s) = 10/s (step of +10 kPa)
  • Setpoint change = 0 (pure regulator problem)
Equations Used & Why

The closed-loop response to disturbance is given by: Y(s)/D(s) = Gd(s) / [1 + Kc·Gp(s)]

Why? In the SR block diagram, the disturbance enters through Gd before the process. After algebra (similar to slides page 123), the disturbance TF has Kc·Gp in the denominator because the controller feedback acts to suppress D. The Gd in the numerator shows how much disturbance "leaks" through to the output.

Final Value Theorem: y(∞) = lim(s→0) s·Y(s) — gives steady-state response.

Initial Value Theorem: y(0⁺) = lim(s→∞) s·Y(s) — gives instantaneous response at t=0.

Calculations

Step 1: Disturbance Transfer Function

Y(s)/D(s) = Gd(s) / [1 + Kc·Gp(s)] = [1/(4s+1)] / {1 + 2×[1/(8s+1)]} = [1/(4s+1)] / [(8s+1+2)/(8s+1)] = [1/(4s+1)] × [(8s+1)/(8s+3)] Y(s)/D(s) = (8s+1) / [(4s+1)(8s+3)]

Step 2: Immediate Effect (t → 0⁺, Initial Value Theorem)

y(0⁺) = lim(s→∞) s · Y(s) = lim(s→∞) s · [(8s+1)/((4s+1)(8s+3))] · (10/s) = lim(s→∞) 10(8s+1) / [(4s+1)(8s+3)] = lim(s→∞) 80s / (32s²) = lim(s→∞) 80/(32s) = 0 Immediate effect = 0 kPa // Both Gd and Gp have lags (τ=4 and τ=8 min) → no instantaneous response

Step 3: Final Pressure After Control (Final Value Theorem)

y(∞) = lim(s→0) s · Y(s) = lim(s→0) s · [(8s+1)/((4s+1)(8s+3))] · (10/s) = lim(s→0) 10(8s+1) / [(4s+1)(8s+3)] = 10 × (0+1) / [(0+1)(0+3)] = 10 × 1 / (1 × 3) = 10/3 y(∞) = 3.33 kPa // Final pressure rises by 3.33 kPa above setpoint — residual offset

Step 4: SSE and Speed of Rejection

Steady-State Error EXISTS = 3.33 kPa // P-only control cannot eliminate disturbance offset // Need PI or PID to get SSE = 0 for disturbance rejection Speed of rejection — CL time constant: Characteristic equation from 1 + KcGp(s) = 0: 1 + 2/(8s+1) = 0 → 8s+1+2 = 0 → 8s+3 = 0 → s = −3/8 τCL = 8/3 = 2.67 min (vs open-loop τ = 8 min) Disturbance is rejected with time constant τCL ≈ 2.67 min // Controller reduces effective process time constant from 8 → 2.67 min (3× faster!)
